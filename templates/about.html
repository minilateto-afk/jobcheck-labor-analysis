資料處理說明 Data Processing

專題定位： 本系統以政府公開違反勞動法令資料為基礎，建立求職前可參考的公開紀錄查詢與趨勢分析系統。

一、專題簡介

本專題名稱為「基於公開違反勞動法令資料之求職參考分析系統」。 系統透過 Python 擷取政府公開資料，並使用 Pandas 進行欄位標準化、 資料清理與違規內容分類，最後以 Flask 網頁呈現公司查詢、摘要卡、 公開紀錄列表與趨勢儀表板。

本系統不直接評價公司好壞，也不建立黑名單，而是將政府已公開之違反勞動法令紀錄整理成較容易閱讀的資料， 作為求職者投履歷或面試前了解公開紀錄的參考。

二、使用情境

使用者可在投履歷或面試前輸入公司名稱，查詢該事業單位是否曾出現在公開違反勞動法令資料中。 系統會整理公開紀錄筆數、最近公告日期、主要違規類型與處分金額，並以摘要卡方式呈現。

三、資料處理流程

政府公開資料 → 欄位標準化 → 資料清理 → 違規分類 → Dashboard 呈現

系統先使用 Requests 下載政府公開 CSV、Excel 或 ODS 資料， 再透過 Pandas 統一欄位名稱，處理日期格式、處分金額與空值。 接著依違規內容關鍵字進行規則式分類，最後輸出成統一 CSV， 供 Flask 頁面呈現查詢結果與統計圖表。

四、欄位標準化

不同縣市資料欄位名稱可能不一致，因此系統建立欄位對應表， 將原始資料轉換為統一格式，方便後續分析。

原始欄位可能名稱 系統統一欄位
事業單位名稱、事業單位或事業主名稱、公司名稱 company_name
主管機關、公告機關、處分機關 city / authority
違反法規法條、違反法令條款、違反條文 violated_law
違反法規內容、違規內容、違反事實 violation_content
處分金額、裁罰金額、罰鍰金額 penalty_amount

五、違規類型分類規則

系統使用規則式關鍵字分類，將法規文字整理成求職者較容易理解的類型。 此分類僅作為資料整理，不代表法律判斷。

分類 常見判斷關鍵字
工時問題 延長工時、出勤紀錄、工作時間、休息時間
薪資／加班費問題 工資、加班費、未全額給付、最低工資
休假問題 例假、休息日、特別休假、國定假日
勞退／保險問題 勞工退休金、提繳、就業保險
職業安全問題 職業安全衛生、職災、防護措施

六、資料限制與使用提醒

本系統僅整理政府公開資料中已公告之違反勞動法令紀錄。 未出現在資料中不代表該事業單位完全沒有勞動爭議； 出現在資料中也不代表可直接判斷公司整體工作環境。 系統分析結果僅作為求職前了解公開紀錄之參考。
