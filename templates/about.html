Project Overview

專案說明：JobCheck 求職公開紀錄分析系統

本專案以勞動部公開違反勞動法令資料為基礎，設計一套可自動下載、整理、查詢與視覺化的求職參考系統。 系統目的不是評斷企業好壞，而是將分散且不易閱讀的公開資料整理成較容易理解的資訊， 協助求職者在投遞履歷前多一項公開紀錄參考。

Data Pipeline

資料處理流程

系統將勞動部公開資料從原始檔案轉換成可查詢、可分類與可視覺化的分析結果。

01

動態下載公開資料

系統啟動時會連線至勞動部公開查詢系統，取得表單 token， 並以 POST 方式依縣市下載公開資料壓縮檔。

02

解壓縮與格式修復

下載後自動解壓縮 zip，並處理官方 CSV 可能出現的表頭位置不固定、 欄位數不一致與中文編碼問題。

03

欄位標準化

將不同縣市資料中的欄位統一整理為公司名稱、公告日期、處分日期、 違反法規、法條敘述與處分金額等標準欄位。

04

資料清理與去重

清理公司名稱、拆分負責人、轉換金額格式，並依公司名稱、日期、 法規與法條敘述移除重複資料。

05

違規類型分類

依關鍵字將資料分類為工時、薪資／加班費、休假、職安、勞退保險與職場平等等類型。

06

查詢與視覺化

透過 Flask 網站提供公司查詢、公開紀錄列表、趨勢儀表板與求職參考摘要。

Label Design

公開紀錄觀察標籤

為避免將公司主觀標籤化，系統不使用「風險分數」或「黑名單」， 而是以公開紀錄筆數與重複出現情形，提供中性的觀察標籤。

Level 0

查無紀錄

目前匯入的公開資料中，未查到符合公司名稱的紀錄。

Level 1

單筆紀錄

查到少量公開紀錄，建議求職者閱讀違反法規與公告日期。

Level 2

多筆紀錄

同一事業單位出現多筆公開紀錄，建議比對是否集中於特定類型或時間。

Level 3

重複出現

同一事業單位多次出現在公開資料中，求職前可特別留意紀錄內容。

System Function

系統功能定位

公司查詢

輸入公司名稱或關鍵字，系統會從已匯入資料中搜尋相關事業單位公開紀錄。

趨勢儀表板

整理違規類型分布、縣市分布、重複出現事業單位與資料來源統計。

資料處理透明化

公開說明資料來源、清理流程、分類方式與系統限制，提升專題可信度。

Limitation

系統限制與改進方向

  • 資料內容以公開公告為準，未出現在資料中的公司不代表一定沒有勞動爭議。
  • 不同縣市下載檔案欄位可能不完全一致，因此系統需進行欄位標準化與缺值處理。
  • 部分縣市可能因外部網站連線狀態暫時下載失敗，系統會保留已成功匯入的資料進行分析。
  • 後續可加入公告日期篩選、公司名稱模糊比對、資料更新時間顯示與更細緻的分類規則。
